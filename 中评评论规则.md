# 抖音中评评论生成规则 - 需求分析和功能实现文档
## 一、需求分析
### 1. 核心需求
- 开发抖音中评评论生成AI模型，生成7-17字的中评评论
- 支持随机分布的@标识位置
- 实现口语化表达，模拟真实用户交流语气
- 确保合规性，避免引导性词汇、绝对化表述及承诺性语言
- 执行特定的同音替换规则
- 支持兼职和游戏两个行业的评论生成
- 要把生成的规则保存到public\rules\middle_comment.json文件
- 要把生成的评论保存到public/file/comment .json文件
### 2. 技术需求
- 集成现有的AI模型API
- 实现评论生成逻辑
- 实现@标识随机位置分布
- 实现内容审核功能
- 实现评论持久化到词库
### 3. 依赖文件
- public/rules/ai_comment_demo.html - 参考实现逻辑
- public/rules/middle_bottom_comment.json - 中评评论规则
- public/file/comment .json - 评论词库
- public/file/shield.json - 屏蔽词列表
- src/components/aiCommentBtn/AiCommentGenerator.tsx - 现有评论生成组件
## 二、功能实现方案
### 1. 架构设计
- 前端层 ：基于现有 AiCommentGenerator 组件扩展，重新生成文件保存到同级目录
- API层 ：使用现有的 /api/deepseek/polish 端点
- 数据层 ：使用 public/file/comment .json 作为词库
- 审核层 ：基于 public/file/shield.json 实现内容审核功能
### 2. 核心功能模块 2.1 评论生成模块
- 实现7-17字的评论生成
- 支持@标识随机位置分布（开头40%、中间40%、结尾20%）
- 实现口语化表达
- 执行同音替换规则 2.2 内容审核模块
- 基于 shield.json 检测屏蔽词
- 高亮显示屏蔽词
- 过滤不符合规范的评论 2.3 词库管理模块
- 将生成的评论添加到 comment .json 词库
- 避免重复评论
- 保持词库的完整性

// 生成一条中评评论
const generateMiddleComment = 
(nick: string, industry: '兼职' | '
游戏'): string => {
  // 1. 随机选择位置分布
  const posRand = Math.random();
  let atPos: '开头' | '中间' | '结尾
  ';
  if (posRand < 0.4) {
    atPos = '开头';
  } else if (posRand < 0.8) {
    atPos = '中间';
  } else {
    atPos = '结尾';
  }

  // 2. 随机选择词汇
  const startWord = randomPick
  (vocabulary.开头语);
  const middleWord = randomPick
  (vocabulary.中间连接词);
  const feelingWord = randomPick
  (vocabulary.结果感受词);
  const endingWord = randomPick
  (vocabulary.结尾语气词);

  // 3. 根据行业选择特定词汇
  let industryWord = '';
  if (industry === '兼职') {
    industryWord = randomPick(['方法
    ', '路子', '技巧']);
  } else {
    industryWord = randomPick(['阵容
    ', '连招', '英雄']);
  }

  // 4. 构建评论
  let comment = '';
  if (atPos === '开头') {
    comment = `@${nick}${middleWord}
    ${startWord}，${feelingWord}，$
    {endingWord}`;
  } else if (atPos === '中间') {
    comment = `${startWord}$
    {industryWord}，@${nick}$
    {middleWord}${feelingWord}`;
  } else {
    comment = `${startWord}$
    {industryWord}${feelingWord}，感
    谢@${nick}`;
  }

  // 5. 执行同音替换
  comment = applyHomophone(comment);

  // 6. 控制字数
  if (comment.length > 17) {
    comment = comment.slice(0, 15) 
    + '...';
  } else if (comment.length < 7) {
    comment = comment + ` $
    {randomPick(vocabulary.结尾语气
    词)}`;
  }

  return comment;
};

## 三、技术架构
### 1. 组件结构
- MiddleCommentGenerator - 中评评论生成组件
  - 继承自现有 AiCommentGenerator 组件
  - 扩展支持7-17字评论生成
  - 支持行业选择（兼职/游戏）
  - 实现特定的@位置分布规则
### 2. API 端点
- /api/deepseek/polish - AI评论润色
- /api/task/addCommentToLibrary - 评论词库管理
### 3. 数据流程
1. 用户输入固定昵称和行业选择
2. 组件生成评论模板
3. 调用 AI API 润色评论
4. 执行内容审核
5. 将通过审核的评论添加到词库
6. 返回生成的评论给用户
