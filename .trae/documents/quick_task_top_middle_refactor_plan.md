# 上中评快捷派单页面重构计划

## 任务目标
重构 `quick-task-top-middle` 页面的UI布局，优化视频链接输入方式和评论生成流程。

## 任务分解

### [ ] 任务1: 重构视频链接输入方式
- **Priority**: P0
- **Depends On**: None
- **Description**:
  - 移除现有的视频链接添加功能
  - 页面加载后默认创建5个视频链接输入表单
  - 每个输入表单独立验证
- **Success Criteria**:
  - 页面加载后显示5个视频链接输入框
  - 每个输入框可以独立输入和验证
- **Test Requirements**:
  - `programmatic` TR-1.1: 页面加载后DOM中存在5个视频链接输入元素
  - `human-judgement` TR-1.2: 输入框布局清晰，易于使用
- **Notes**: 保持输入框的样式一致性

### [ ] 任务2: 实现视频链接校验逻辑
- **Priority**: P0
- **Depends On**: 任务1
- **Description**:
  - 视频链接长度必须大于35个字符
  - 视频链接必须包含"抖音"或"douyin"字符
  - 实时显示校验结果
- **Success Criteria**:
  - 输入不符合要求时显示错误提示
  - 输入符合要求时显示成功提示
- **Test Requirements**:
  - `programmatic` TR-2.1: 输入长度<=35时显示错误提示
  - `programmatic` TR-2.2: 输入不包含"抖音"或"douyin"时显示错误提示
  - `programmatic` TR-2.3: 输入符合要求时显示成功提示
- **Notes**: 校验逻辑应该实时执行，提供即时反馈

### [ ] 任务3: 迁移AI生成评论功能
- **Priority**: P0
- **Depends On**: None
- **Description**:
  - 移除独立的"AI生成评论"按钮
  - 将AI生成评论功能集成到"发布任务"按钮中
  - 固定使用快速生成模式，行业随机调用
- **Success Criteria**:
  - 页面上不存在独立的"AI生成评论"按钮
  - 点击"发布任务"按钮时先调用AI生成评论
- **Test Requirements**:
  - `programmatic` TR-3.1: 页面DOM中不存在"AI生成评论"按钮
  - `human-judgement` TR-3.2: 点击"发布任务"按钮时先显示评论生成过程
- **Notes**: 确保AI生成评论失败时给出清晰的错误提示

### [ ] 任务4: 修改任务发布次数逻辑
- **Priority**: P1
- **Depends On**: 任务2
- **Description**:
  - 任务发布次数根据有效视频链接的数量自动计算
  - 移除手动输入发布次数的功能
- **Success Criteria**:
  - 任务发布次数显示为有效视频链接的数量
  - 无法手动修改发布次数
- **Test Requirements**:
  - `programmatic` TR-4.1: 任务发布次数自动更新为有效视频链接数量
  - `programmatic` TR-4.2: 发布次数输入框变为只读或被移除
- **Notes**: 确保发布次数始终与有效视频链接数量一致

### [ ] 任务5: 强化发布任务验证
- **Priority**: P1
- **Depends On**: 任务2, 任务4
- **Description**:
  - 发布任务时必须至少有一条有效的视频链接
  - 验证评论内容是否完整
- **Success Criteria**:
  - 没有有效视频链接时点击"发布任务"显示错误提示
  - 评论内容不完整时显示错误提示
- **Test Requirements**:
  - `programmatic` TR-5.1: 无有效视频链接时点击"发布任务"显示错误提示
  - `programmatic` TR-5.2: 评论内容为空时显示错误提示
- **Notes**: 验证逻辑应该在AI生成评论之前执行

### [ ] 任务6: 优化UI布局和用户体验
- **Priority**: P2
- **Depends On**: 任务1, 任务2, 任务3, 任务4
- **Description**:
  - 调整页面布局，使各个模块更加清晰
  - 优化表单验证提示的显示方式
  - 确保页面在不同设备上的响应式显示
- **Success Criteria**:
  - 页面布局美观，模块划分清晰
  - 验证提示位置合理，易于理解
  - 页面在移动端和桌面端都能正常显示
- **Test Requirements**:
  - `human-judgement` TR-6.1: 页面布局美观，模块划分清晰
  - `human-judgement` TR-6.2: 验证提示位置合理，易于理解
  - `human-judgement` TR-6.3: 页面在移动端和桌面端都能正常显示
- **Notes**: 保持与其他页面的设计风格一致

## 实现步骤

1. **准备工作**:
   - 备份原始代码
   - 分析当前代码结构和功能

2. **实现任务1和任务2**:
   - 修改视频链接输入方式
   - 实现视频链接校验逻辑

3. **实现任务3**:
   - 移除AI生成评论按钮
   - 集成AI生成评论功能到发布任务按钮

4. **实现任务4**:
   - 修改任务发布次数逻辑
   - 确保发布次数与有效视频链接数量一致

5. **实现任务5**:
   - 强化发布任务验证
   - 确保所有必填项都已完成

6. **实现任务6**:
   - 优化UI布局和用户体验
   - 测试页面在不同设备上的显示效果

7. **测试和调试**:
   - 测试所有功能是否正常工作
   - 修复可能存在的bug

8. **完成重构**:
   - 验证所有功能都已实现
   - 确保代码质量和性能

## 预期成果

- 页面加载后默认显示5个视频链接输入表单
- 视频链接输入时实时进行有效性校验
- 点击"发布任务"按钮时先调用AI生成评论
- 任务发布次数根据有效视频链接数量自动计算
- 页面布局美观，用户体验良好

## 风险评估

- **风险1**: AI生成评论可能会失败，需要添加错误处理
- **风险2**: 视频链接校验逻辑可能会影响用户体验，需要确保提示清晰
- **风险3**: 自动计算发布次数可能会让用户感到困惑，需要添加说明

## 应对措施

- **风险1应对**: 添加AI生成评论失败的错误处理和提示
- **风险2应对**: 优化校验提示的显示方式，确保提示清晰易懂
- **风险3应对**: 在发布次数显示区域添加说明文字，解释发布次数的计算方式
