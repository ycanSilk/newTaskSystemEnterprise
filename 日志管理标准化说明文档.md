



## 一、页面日志、提示和审计记录标准化说明文档

### 1. 适用范围

本标准化说明文档适用于项目中所有需要与API交互的页面，包括但不限于：
- 登录页面
- 注册页面
- 任务发布页面
- 用户管理页面
- 数据统计页面

### 2. 标准化结构

#### 2.1 日志功能

**2.1.1 日志文件结构**
```
src/
├── logs/
│   ├── [模块名]/
│   │   ├── [页面名]/
│   │   │   ├── api-request.log    # API请求日志
│   │   │   ├── api-error.log      # API错误日志
│   │   │   └── js.log             # JavaScript错误日志
├── audit/
│   ├── [模块名]/
│   │   ├── [页面名]/
│   │   │   └── audit.log          # 审计日志
```

**2.1.2 日志功能实现**
1. **时间处理**：使用北京时间格式化日期
2. **目录确保**：确保日志目录存在，避免写入失败
3. **日志写入**：将日志写入指定文件，确保日志持久化
4. **日志分类**：根据日志类型写入不同文件，便于管理

**2.1.3 日志内容规范**
- **API请求日志**：包含时间戳、请求类型、端点、方法、参数
- **API错误日志**：包含时间戳、错误类型、端点、方法、错误详情
- **JavaScript错误日志**：包含时间戳、错误类型、上下文、错误信息、堆栈信息
- **审计日志**：包含时间戳、操作类型、操作动作、详细信息

#### 2.2 错误处理

**2.2.1 错误码映射**
- 定义统一的错误码映射表，确保错误信息一致
- 为每种错误码提供用户友好的错误提示
- 区分用户级别和技术级别的错误信息

**2.2.2 错误处理流程**
1. **响应状态检查**：检查API响应状态，确保响应正常
2. **响应格式检查**：检查响应是否为JSON格式，处理非JSON响应
3. **错误码处理**：根据错误码返回对应的错误信息
4. **用户提示**：显示用户友好的错误提示
5. **日志记录**：记录详细的错误信息，便于排查

#### 2.3 审计功能

**2.3.1 审计记录点**
- **页面加载**：记录页面加载事件，包含用户代理信息和屏幕尺寸
- **用户操作**：记录用户的关键操作，如按钮点击、表单提交等
- **API调用**：记录API调用的结果，包括成功和失败
- **错误处理**：记录错误处理的过程和结果

**2.3.2 审计日志内容**
- **时间戳**：使用北京时间，确保时间一致性
- **操作类型**：如登录、注册、发布任务等
- **操作动作**：如开始、成功、失败等
- **详细信息**：包含操作的具体信息，如用户名、操作结果等

#### 2.4 代码结构

**2.4.1 模块划分**
1. **工具函数**：时间处理、目录确保、日志写入等
2. **日志函数**：API请求日志、API错误日志、JavaScript错误日志、审计日志
3. **错误处理函数**：错误码处理、用户友好消息生成
4. **业务逻辑函数**：表单提交、API调用等

**2.4.2 代码风格**
- **命名规范**：使用驼峰命名法，函数名清晰表达功能
- **注释规范**：为关键函数和逻辑添加注释，提高代码可读性
- **错误处理**：使用try-catch捕获错误，确保代码健壮性
- **日志记录**：在关键节点添加日志记录，便于排查问题

### 3. 实施步骤

**3.1 准备工作**
1. 确定页面的模块名和页面名，创建对应的日志目录
2. 定义页面的错误码映射表，确保错误信息一致
3. 确定页面的审计记录点，确保所有关键操作都有审计记录

**3.2 代码实现**
1. 添加日志功能相关的工具函数和日志函数
2. 添加错误处理相关的函数和逻辑
3. 在关键节点添加审计日志记录
4. 增强错误处理，确保代码健壮性

**3.3 测试验证**
1. 测试页面的各种操作，确保日志正确记录
2. 测试错误场景，确保错误处理正确
3. 验证审计日志的完整性和准确性
4. 确保页面在各种场景下都能正常运行

### 4. 最佳实践

- **日志分级**：根据日志类型和级别进行分类，便于管理和查询
- **错误处理**：全面的错误处理，确保系统在各种情况下都能正常运行
- **审计记录**：完整的审计记录，便于监控用户行为和系统状态
- **代码质量**：优化代码结构和风格，提高代码的可读性和可维护性
- **用户体验**：提供友好的错误提示，提高用户体验
- **性能优化**：避免在关键路径上记录日志，影响系统性能
- **日志记录**：只在关键节点添加日志记录，便于排查问题

### 5. 总结

通过实施本标准化说明文档，项目中的所有页面都将具备完善的日志管理、错误处理和审计功能，提高系统的可维护性、安全性和用户体验。同时，标准化的代码结构和实现方式，将有助于团队协作和代码维护，确保项目的长期稳定发展。
